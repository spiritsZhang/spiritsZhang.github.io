项目说明

这里展示我亲自完成并能够讲清楚设计、实现与取舍的项目。每个项目都将尽量提供源码、文档、演示或技术复盘，确保内容可验证。

个人技术博客

项目背景

搭建一个长期维护的技术内容平台，用于整理学习记录、项目复盘，并向面试官集中展示个人技术成长。

技术方案

  • 静态站点生成： Hexo
  • 界面主题： Butterfly
  • 内容格式： Markdown
  • 托管与部署： GitHub Pages
  • 版本管理： Git / GitHub

已完成工作

  • 完成站点初始化、主题配置和 GitHub Pages 部署。
  • 建立项目、文章、分类、标签和个人介绍等信息结构。
  • 启用文章目录、代码高亮、深色模式和响应式页面。
  • 优化站点元信息、导航结构和面向校招的内容表达。

项目链接

  • 在线访问
  • 查看源码

下一项作品

新的项目会在达到以下标准后加入：

  1. 具备明确的问题背景和目标。
  2. 能说明个人职责与关键技术决策。
  3. 有可运行代码、演示结果或完整技术文档。
  4. 能复盘遇到的问题、解决过程和改进方向。

目前正在整理中。相比堆砌不完整的练习项目，我会优先展示少量但能够深入讲解的作品。
